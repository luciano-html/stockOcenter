Eres un arquitecto de software senior. Necesito diseñar y construir una app web para gestionar stock de componentes de sillas.

## Contexto del negocio
- Fabricamos sillas compuestas por componentes: cilindro, ruedas, estrella, tapizado, espuma, asiento interior, asiento exterior, respaldo, reposabrazos, mecanismo, base, tornillería
- Cada tipo de silla tiene una **lista de materiales (BOM)** que define cuántas unidades de cada componente necesita

## Reglas de negocio
1. **Ingreso de stock**: cuando llega un camión, se cargan las cantidades recibidas de cada componente
2. **Órdenes de trabajo (OT)**: se crean para fabricar N sillas de un tipo específico
3. **Consumo de stock**: cuando una OT se marca como "finalizada", se descuenta el BOM correspondiente del stock
4. **Cálculo de sillas posibles**: el sistema debe calcular en tiempo real cuántas sillas de cada tipo se pueden armar con el stock actual (min(componente / cantidad_por_silla))

## Stack técnico
- **Frontend**: React + TypeScript + Vite + react-hook-form + zod + TanStack Query
- **Backend**: Node + Express + TypeScript (MVC)
- **BD**: MongoDB + Mongoose
- Skill: node-express-mongo-react (Stack Definido por el Dev)

## Requerimientos
- CRUD de componentes, tipos de silla y BOM
- CRUD de órdenes de trabajo con estados (pendiente, en_progreso, finalizada)
- Vista de stock actual con alertas de componentes críticos (stock < mínimo)
- Cálculo automático de sillas fabricables
- Historial de movimientos de stock (ingresos y egresos)
